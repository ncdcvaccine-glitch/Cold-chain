# منصة جرد معدات سلسلة التبريد (101 بلدية)

منصة ويب لجرد معدات سلسلة التبريد (الثلاجات، الفريزرات، صناديق التبريد...) الخاصة ببرنامج
التطعيم على مستوى 101 بلدية في ليبيا. تعتمد على **Firebase** (Authentication + Firestore)
ويمكن استضافتها مجاناً عبر **Firebase Hosting** أو **GitHub Pages**.

## المستخدمون
- **مشرف بلدية (supervisor)**: يسجّل دخول بحساب خاص ببلديته فقط، يضيف المراكز الصحية
  ومعدات التبريد الخاصة بكل مركز (النوع، الماركة، الرقم التسلسلي، **السعة التخزينية باللتر**،
  الحالة، ملاحظات).
- **إدارة (admin)**: تسجّل دخول بحساب واحد، تشاهد بيانات **كل** البلديات، تصفّيها، وتصدّرها
  إلى ملف **Excel**.

## هيكل المشروع
```
index.html         صفحة تسجيل الدخول (توجّه تلقائياً حسب الدور)
supervisor.html     واجهة مشرف البلدية (إدخال البيانات)
admin.html          لوحة الإدارة (عرض + تصفية + تصدير Excel)
firebase-config.js  إعدادات Firebase (عدّلها ببيانات مشروعك)
data.js             قائمة البلديات الـ101 وأنواع المعدات وحالاتها
styles.css          التنسيق العام
firestore.rules     قواعد أمان Firestore (دور كل مستخدم + بلديته)
seed-users.js        سكريبت لإنشاء حسابات الدخول للبلديات دفعة واحدة
manifest.json        إعدادات تطبيق الويب التقدّمي (PWA) — اسم/أيقونة/ألوان التطبيق
service-worker.js    تخزين مؤقت يجعل واجهة التطبيق تفتح ولو الاتصال ضعيف
icons/               أيقونات التطبيق (192px / 512px + نسخ maskable)
```

## التطبيق على الهاتف (PWA) — قابل للتثبيت بدون متجر تطبيقات
المنصة مهيأة بالكامل كـ **Progressive Web App**: بعد نشرها على رابط HTTPS (Firebase Hosting
أو GitHub Pages، كلاهما HTTPS تلقائياً)، يمكن لأي مستخدم تثبيتها كأيقونة تطبيق حقيقية على
هاتفه:

- **أندرويد (Chrome):** عند فتح الرابط ستظهر رسالة "تثبيت التطبيق" أسفل الشاشة تلقائياً
  (مُفعّلة في `index.html`)، أو يدوياً من قائمة المتصفح (⋮) > **"تثبيت التطبيق"** /
  **"إضافة إلى الشاشة الرئيسية"**.
- **آيفون (Safari):** افتح الرابط في Safari ثم اضغط على زر المشاركة (⬆) واختر
  **"إضافة إلى الشاشة الرئيسية"** (Safari لا يدعم نافذة التثبيت التلقائية لكنه يثبّت
  التطبيق بنفس الطريقة).

بعد التثبيت يفتح التطبيق بدون شريط عنوان المتصفح، بأيقونته الخاصة، ويعمل بشكل أساسي حتى
مع ضعف الاتصال (بفضل `service-worker.js`)، علماً أن البيانات الفعلية (الإدخال والعرض)
تبقى تتطلب اتصال إنترنت للتواصل مع Firebase.

> ⚠️ شرط أساسي: يجب أن يكون الموقع منشوراً عبر **HTTPS** (وليس مجرد فتح الملفات محلياً)
> حتى يعمل التثبيت و service worker. كل من Firebase Hosting و GitHub Pages يوفران ذلك
> تلقائياً ومجاناً.


## 1) إعداد مشروع Firebase
1. اذهب إلى https://console.firebase.google.com وأنشئ مشروعاً جديداً.
2. فعّل **Authentication > Sign-in method > Email/Password**.
3. أنشئ قاعدة بيانات **Firestore Database** (وضع Production).
4. من **Project Settings > Your apps**، أضف تطبيق ويب (</>) وانسخ بيانات الإعداد
   (apiKey, authDomain...) إلى ملف `firebase-config.js` بدلاً من القيم النموذجية `YOUR_...`.
5. من **Firestore > Rules**، الصق محتوى ملف `firestore.rules` ثم انشر القواعد.

## 2) إنشاء حسابات الدخول (101 بلدية + إدارة)
الطريقة الأسهل: استخدم `seed-users.js` (يعمل مرة واحدة من جهازك، ليس جزءاً من الموقع):
```bash
npm install firebase-admin
# نزّل مفتاح حساب الخدمة (Service Account) من إعدادات المشروع وضعه باسم serviceAccountKey.json
node seed-users.js
```
سيُنشئ السكريبت تلقائياً:
- حساب إدارة: `admin@coldchain.ly`
- حساب لكل بلدية بصيغة: `m1@coldchain.ly`, `m2@coldchain.ly` ... حتى `m101@coldchain.ly`
  (مع طباعة كلمة مرور عشوائية لكل حساب — **احفظها فوراً** لأنها لن تظهر مرة أخرى)

> بديل: يمكنك إنشاء الحسابات يدوياً من **Authentication > Users > Add user**، ثم إضافة
> وثيقة لكل مستخدم في مجموعة `users` بصيغة:
> `{ role: "supervisor", municipality: "اسم البلدية", displayName: "..." }`
> أو `{ role: "admin", displayName: "..." }` لحساب الإدارة.

## 3) رفع المشروع على GitHub
```bash
cd coldchain-inventory
git init
git add .
git commit -m "إطلاق منصة جرد معدات سلسلة التبريد"
git branch -M main
git remote add origin https://github.com/<اسم-المستخدم>/<اسم-المستودع>.git
git push -u origin main
```

## 4) النشر (اختر إحدى الطريقتين)

### أ) Firebase Hosting (موصى به)
```bash
npm install -g firebase-tools
firebase login
firebase init hosting   # اختر مجلد المشروع كـ public directory
firebase deploy
```

### ب) GitHub Pages
من إعدادات المستودع على GitHub: **Settings > Pages > Source: Deploy from branch (main)**.
بعد بضع دقائق سيصبح الموقع متاحاً على رابط من نوع:
`https://<اسم-المستخدم>.github.io/<اسم-المستودع>/`

## ملاحظات أمان مهمة
- لا تضع مفتاح حساب الخدمة (`serviceAccountKey.json`) في GitHub — أضفه إلى `.gitignore`.
- `firebase-config.js` يحتوي على مفاتيح عامة (Web API Key) آمنة للنشر؛ الحماية الفعلية
  تأتي من **Firestore Rules** المرفقة، تأكد من نشرها قبل تفعيل المنصة.
- يمكنك لاحقاً تغيير كلمات مرور البلديات من **Authentication > Users**.

## التوسعة المستقبلية (اختياري)
- إضافة صلاحية "تعديل" للمعدات للمشرف (حالياً إضافة وحذف فقط).
- إشعارات تلقائية عند معدات بحالة "خارج الخدمة".
- تصدير تقرير لكل بلدية على حدة من لوحة الإدارة.
